## Overview

Onboarding Components take all the magic that resides within the Footprint embedded onboarding and exposes them as individual web/React components you can embed, fully stylize, and control inside your product.

It's a set of unstyled, composable components that can be used to build custom UIs on top of Footprint's APIs.

## Why use Onboarding Components?

Use Onboarding Components if you want more control over the styles and layout of your UI while leveraging Footprint's APIs.
If the appearance options aren’t enough, Onboarding Components might be what you need. Plus, they work seamlessly alongside our prebuilt components.

**Component-first Approach** - Building custom UIs with Footprint is as easy as using our prebuilt components. Onboarding Components manage the business logic for you, so you just need to focus on the UI.

**Unstyled, with Magic** - Create great authentication flows using top-notch web components.

## Getting started

**Onboarding Components currently only works with React**. To get started, install Footprint packages:

```bash
# With NPM
npm install @onefootprint/footprint-js
npm install @onefootprint/footprint-react@beta

# With yarn
yarn add @onefootprint/footprint-js
yarn add @onefootprint/footprint-react@beta
```
