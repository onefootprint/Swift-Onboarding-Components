# Footprint Secure Form

Our embedded form component can help you collect sensitive PII data in your product in a PCI compliant way. Below is a guide that describes how to integrate our footprint form component to collect user data that automatically gets saved to the user's Footprint vault.

<img alt="footprint form" height={250} src="/form/form.png" width={720} />

## Add Footprint.js to your app

1. Go to Footprint dashboard [Settings - Roles](https://dashboard.onefootprint.com/settings?tab=roles) page. Create a new role for card vaulting.

   - Select `Create and update users` from the permissions list.

2. Go to Footprint dashboard [Developer - Secret API Keys](https://dashboard.onefootprint.com/developers?tab=api_keys) page. Create a new Secret API Key with the role you have just created.

   - Grab the Secret API Key, for example `sk_live_Zlkert4344in3Gerfere`.

<inline-alert variant='warning'>Treat this key like a sensitive password! Your API keys carry many privileges, so be sure to keep them secure! Do not post your secret keys in publicly accessible places like GitHub or client-side code like web front-ends.</inline-alert>

4. For the user you will show the component to, if there is no existing Footprint user vault, create a new vault using the `POST /users` API by making a call from the backend. More details can be found [here](/articles/integrate/migrate-existing-data#step-1-create-a-new-user-vault). Grab the Footprint user ID, for example `fp_id_ItdeKyRaEJ7Zpao4i6vTbN`.

5. Generate an auth token for collecting data from the backend

   - You will need to specify a unique `card_alias` that the data will be stored under since Footprint user vaults support multiple cards. You can find more information on the card aliases [here](/articles/vault/apis#payment-cards).
   - You will need to provide a full list of fields you intend to collect. In the example below, card number, CVC, expiration and billing address zip and country. These fields drive the inputs shown on the form.
   - You will also need to specify a time-to-live (TTL) between 1 min and 1 day.

```bash
curl https://api.onefootprint.com/users/fp_id_ItdeKyRaEJ7Zpao4i6vTbN/client_token \
      -X POST \
      -u sk_live_Zlkert4344in3Gerfere: \
      -d '{
            "fields": [
              "card.card_alias.name",
              "card.card_alias.number",
              "card.card_alias.cvc",
              "card.card_alias.expiration",
              "card.card_alias.billing_address.zip",
              "card.card_alias.billing_address.country"
            ],
            "scopes": [
              "vault"
            ],
            "ttl": 1800
        }'
```

A sample response can be found below:

```bash
{"token":"cttok_joXzzB0kIVW0fMCB7RWPAHWt8itWdFWpit","expires_at":"2023-07-20T03:59:03.579654Z"}
```

The token returned will need to be passed into the Footprint component on the frontend.

6. Add `footprint.js` to your app.

```bash
# With NPM
npm install @onefootprint/footprint-js

# With yarn
yarn add @onefootprint/footprint-js
```

or

```html
<head>
  <link
    rel="stylesheet"
    href="https://unpkg.com/@onefootprint/footprint-js@latest/dist/footprint-js.css"
  />
</head>
<body>
  ...
  <script
    crossorigin
    src="https://unpkg.com/@onefootprint/footprint-js@latest/dist/footprint-js.umd.js"
  ></script>
</body>
```

7. Embed the Form component.

Make sure to pass the auth token generated from Step 5.

```typescript
import '@onefootprint/footprint-js/dist/footprint-js.css';
import footprint from '@onefootprint/footprint-js';

const renderForm = () => {
  const component = footprint.init({
    kind: 'form',
    variant: 'inline',
    containerId: 'myContainerId', // unique ID for the div that the form should be rendered in
    authToken: 'cttok_joXzzB0kIVW0fMCB7RWPAHWt8itWdFWpit',
    onComplete: () => {
      // TODO:
    },
  });
  component.render();
};
```

If you are rendering the secure form in an inline variant, it will take up all of the space provided by its parent container. The easiest way to customize the placement is to wrap the it in a div and apply styles to this div. You can also render it in `modal` or `drawer` variants. If you are rendering the secure form in a modal or drawer variant, it will take up the whole viewport when launched.

You can find integration instructions for [Javascript/Typescript](/articles/sdks/javascript#available-components-form-component-integration) and [React](/articles/sdks/react#available-components-footprint-form-integration), in our docs as well.

## Decrypt or Proxy card data from the backend

When the user has completed filling the secure collection form on your product, you can decrypt the data collected. More details can be found [here](/articles/vault/apis#decrypt-data-from-a-users-vault)

## Available Form Props

| Variable     | Description                                                                                                                                                                                                                                                           |
| ------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `title`      | Optional. A form header.                                                                                                                                                                                                                                              |
| `authToken`  | Optional. A valid string authentication token generated using the Secret API Key, Footprint User ID and fields to collect. More info [here](/articles/integrate/user-specific-sessions).                                                                                       |
| `onComplete` | Triggered after user completes the form and data is saved to the user vault.                                                                                                                                                                                          |
| `onError`    | Optional. A function that is called there was an unrecoverable error while initializing the onboarding flow. It takes in an error string argument with more details.                                                                                                  |
| `onCancel`   | Triggered when the user abandons the flow. This can be triggered when the user clicks on close or cancel buttons inside our iframe                                                                                                                                    |
| `onClose`    | Triggered when the user abandons the flow. This can be triggered when the user clicks on close or cancel buttons inside our iframe                                                                                                                                    |
| `options`    | Optional. An options object that customizes the flow, can accept `hideButtons`, `hideCancelButton` and `hideFootprintLogo` boolean entries.                                                                                                                           |
| `appearance` | Optional. A `FootprintAppearance` object that customizes the look of your integration                                                                                                                                                                                 |
| `variant`    | Optional. One of `modal`, `inline` or `drawer`.                                                                                                                                                                                                                       |
| `getRef`     | Optional. An async callback that takes one argument and returns a promise. The `ref` object contains a method `save()` that can be called to save the form data (`ref.save()`). Promise is resolved or rejected with a string error when saving to vault is complete. |
