# Footprint.js Library

Our `footprint-js` library lets you integrate your application with Footprint. It can be used together
with any framework or library, such as Angular, React, Vue, or simply plain Javascript. Under the hood, it
launch iframes from your website, and exchange secure messages between Footprint and your application.

# Available Components

Our SDKs support a variety of component kinds.

| Component | Description                                                                                                       |
| --------- | ----------------------------------------------------------------------------------------------------------------- |
| `verify`  | Our onboarding flow that launches in a modal or drawer.                                                           |
| `form`    | A PCI-compliant form to collect user data for specific fields. It can be rendered inline, in a modal or a drawer. |
| `auth`    | Our authentication flow that launches in a modal or drawer.                                                       |

You will find more information on each of these below:

## Installation

1. Install `footprint-js` :

```bash
# With NPM
npm install @onefootprint/footprint-js

# With yarn
yarn add @onefootprint/footprint-js
```

<inline-alert variant='info'>Make sure your application's permissions and content security policies are configured correctly to allow the Footprint integration. More info can be found [here](/integrate/content-security-policy).</inline-alert>

In your implementation, you will need to import both of the following files:

```javascript
import '@onefootprint/footprint-js/dist/footprint-js.css';
import footprint from '@onefootprint/footprint-js';
```

## Verify Component Integration

This is our onboarding flow that can be rendered in a modal or drawer.

1. Grab the Onboarding Publishable Key, for example, `pb_test_VMooXd04EUlnu3AvMYKjMW`.

2. You can trigger Footprint when a custom button is clicked, passing the playbook key and the callback functions:

```javascript
import '@onefootprint/footprint-js/dist/footprint-js.css';
import footprint from '@onefootprint/footprint-js';

const handleButtonClick = () => {
  const component = footprint.init({
    kind: 'verify',
    publicKey: 'pb_test_WNgSBRR7uxoT8JRDBBflgw',
    onComplete: validationToken => {
      console.log(validationToken);
    },
  });
  component.render();
};
```

3. Once the user completes the flow, you'll receive the `validationToken` through the `onComplete` fuction defined on `window.footprintCallbacks`. From there,
   you can post it to your backend.

Click [here](https://github.com/onefootprint/examples/tree/master/idv/frontend-vanilla-js) to check out a full example.

### Usage with frameworks

Let's imagine your app was written using [Vue](https://vuejs.org/) (could be any framework or library). To integrate with Footprint,
you can render a Footprint verify button that launches the onboarding flow when clicked. You can either pass the props directly to the container div, or call our `init` and `render` methods on footprint to launch the verification dialog.

```vue
<script>
import '@onefootprint/footprint-js/dist/footprint-js.css';
import footprint from '@onefootprint/footprint-js';
</script>

<template>
  <div
    data-footprint
    data-kind="verify"
    data-props='{"publicKey":"pb_test_WNgSBRR7uxoT8JRDBBflgw"}'
  ></div>
</template>
```

Alternatively, you can render a button and initialize the onboarding flow in the click callback. To do that, just call the `init` (with any props) and `render` methods on footprint in the event handler.

```vue
<scrip>
import '@onefootprint/footprint-js/dist/footprint-js.css';
import footprint from '@onefootprint/footprint-js';

export default {
  mounted() {
    const button = document.getElementById('my-button');
    button.addEventListener('click', this.handleClick);
  },

  methods: {
    handleClick() {
      const component = footprint.init({
        kind: 'verify',
        publicKey: 'pb_test_WNgSBRR7uxoT8JRDBBflgw',
      });
      component.render();
    },
  },
};
</scrip>
```

Click [here](https://github.com/onefootprint/examples/tree/master/idv/frontend-vite-vue) to check out a full example.

### Listening to events

Footprint provides some events based on some actions performed by the user. To listen to event, just pass it from the `init` method.

```typescript
const component = footprint.init({
  kind: 'verify',
  publicKey: 'pb_test_WNgSBRR7uxoT8JRDBBflgw',
  onComplete: validationToken => {
    console.log(validationToken);
  },
});
component.render();
```

### Available Props

| Variable     | Description                                                                                                                                                                                                                     |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `publicKey`  | Optional. Onboarding configuration public key.                                                                                                                                                                                  |
| `authToken`  | Optional. A valid string authentication token generated using the Secret API Key, Footprint User ID and fields to collect. One of publicKey or authToken must be provided. More info [here](/integrate/user-specific-sessions). |
| `onComplete` | Triggered after completing the onboarding flow. You'll receive a `validationToken`, and from there you can post to your backend                                                                                                 |
| `onError`    | Optional. A function that is called there was an unrecoverable error while initializing the onboarding flow. It takes in an error string argument with more details.                                                            |
| `onCancel`   | Triggered when the user abandons the flow. This can be triggered when the user clicks on the close button inside our iframe                                                                                                     |
| `onClose`    | Triggered when the user closes the flow (either completed or canceled).                                                                                                                                                         |
| `userData`   | Optional. An object that contains previously collected user data that can bootstrap the onboarding flow. More information [here](https://docs.onefootprint.com/integrate/user-data).                                            |
| `options`    | Optional. An options object that customizes the flow, can accept `showLogo` and `showCompletionPage` entries. More info [here](https://docs.onefootprint.com/integrate/customization#available-options)                         |
| `appearance` | Optional. A `FootprintAppearance` object that customizes the look of your integration                                                                                                                                           |
| `variant`    | Optional. One of `modal` or `drawer`. Specifies how the flow will be rendered.                                                                                                                                                  |
| `l10n`       | Optional. Specifies the desired localization. More information [here](https://docs.onefootprint.com/integrate/customization#localization-configuration).                                                                        |

## Form Component Integration

1. Perform steps 1-5 from [here](https://docs.onefootprint.com/embedded-components/getting-started#add-footprint-js-to-your-app).

2. Import the styles file and embed the Footprint component, passing the Auth token, callback functions and any other options:

```typescript
import '@onefootprint/footprint-js/dist/footprint-js.css';
import footprint from '@onefootprint/footprint-js';

const renderForm = () => {
  const component = footprint.init({
    kind: 'form',
    variant: 'inline',
    containerId: 'myContainerId', // unique ID for the div that the form should be rendered in
    authToken: 'utok_123',
    onComplete: () => {
      // TODO:
    },
  });
  component.render();
};
```

3. When the user has completed filling the secure collection form on your product, you can decrypt the data collected. More details can be found [here](https://docs.onefootprint.com/vault/apis#decrypt-data-from-a-users-vault)

Click [here](https://github.com/onefootprint/examples/tree/master/form/frontend-vanilla-js) to check out a full example.

### Usage with frameworks

Let's imagine your app was written using [Vue](https://vuejs.org/) (could be any framework or library).

You can render the Footprint form using the `init` and `render` methods:

```vue
<script>
import '@onefootprint/footprint-js/dist/footprint-js.css';
import footprint from '@onefootprint/footprint-js';

export default {
  mounted() {
    const component = footprint.init({
      kind: 'form',
      variant: 'inline', // alternatively, you can set the variant to 'modal' or 'drawer'
      containerId: 'my-form', // required if rendering "inline"
      authToken: 'utok_joXzzB0kIVW0fMCB7RWPAHWt8itWdFWpit', // auth token generated using the Secret API Key on step 5
      onComplete: handleComplete,
    });
    component.render();
  },
  methods: {
    handleComplete() {
      // TODO: Decrypt or proxy the saved data from your backend
    },
  },
};
</script>

<template>
  <div id="my-form"></div>
</template>
```

Click [here](https://github.com/onefootprint/examples/tree/master/form/frontend-vite-vue) to check out a full example.

### Listening to Form events

Our Form component provides some events based on actions performed by the user. To listen to event, pass callbacks into the props attribute.

```typescript
const component = footprint.init({
  kind: 'form'
  authToken,
  title: 'Add a New Card',
  variant: 'inline',
  containerId: 'my-form',
  onSave: handleSave,
  onClose: handleClose,
  onCancel: handleCancel
});
component.render();
```

| Name | Description |
| ---- | ----------- |

### Available Props

| Variable      | Description                                                                                                                                                                                                                                                           |
| ------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --- |
| `title`       | Optional. A form header.                                                                                                                                                                                                                                              |
| `authToken`   | Optional. A valid string authentication token generated using the Secret API Key, Footprint User ID and fields to collect. More info [here](/integrate/user-specific-sessions).                                                                                       |
| `onComplete`  | Triggered after user completes the form and data is saved to the user vault.                                                                                                                                                                                          |
| `onError`     | Optional. A function that is called there was an unrecoverable error while initializing the onboarding flow. It takes in an error string argument with more details.                                                                                                  |
| `onCancel`    | Triggered when the user abandons the flow. This can be triggered when the user clicks on close or cancel buttons inside our iframe                                                                                                                                    |
| `onClose`     | Triggered when the user abandons the flow. This can be triggered when the user clicks on close or cancel buttons inside our iframe                                                                                                                                    |
| `options`     | Optional. An options object that customizes the flow, can accept `hideButtons`, `hideCancelButton` and `hideFootprintLogo` boolean entries.                                                                                                                           |
| `appearance`  | Optional. A `FootprintAppearance` object that customizes the look of your integration                                                                                                                                                                                 |
| `variant`     | Optional. One of `modal`, `inline` or `drawer`. If `inline`, requires a `containerId`. Specifies how the form will be rendered.                                                                                                                                       |
| `containerId` | Required if `variant` is `inline`. The ID for the HTML element that the form will be rendered in.                                                                                                                                                                     |
| `getRef`      | Optional. An async callback that takes one argument and returns a promise. The `ref` object contains a method `save()` that can be called to save the form data (`ref.save()`). Promise is resolved or rejected with a string error when saving to vault is complete. |     |

## Browser support

Footprint libraries supports all recent versions of major browsers. For the sake of security and providing the best experience to the majority of customers, we do not support less popular browsers that no longer receive security updates.

| Name                    | Versions              |
| ----------------------- | --------------------- |
| Chrome                  | Last 3 major versions |
| Chrome Mobile           | Last 3 major versions |
| Firefox                 | Last 3 major versions |
| Microsoft Edge          | Last 3 major versions |
| Safari OSX              | Last 3 major versions |
| Safari iOS              | Last 3 major versions |
| Android default browser | Last 3 major versions |

Footprint products depend on certain native browser functionality. Polyfilling certain browser features (e.g Promises API) may prevent footprint.js from working normally. If footprint.js isn't functioning as expected, try removing polyfills to determine if they might be causing the issue.
