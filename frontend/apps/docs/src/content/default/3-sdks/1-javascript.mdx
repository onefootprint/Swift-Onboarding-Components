## Footprint libraries

Our `footprint-js` and `footprint-components-js` libraries let you integrate your application with Footprint. They can be used together
with any framework or library, such as Angular, React, Vue, or simply plain Javascript. Under the hood, they
launch iframes from your website, and exchange secure messages between Footprint and your application.

### Browser support

Footprint libraries supports all recent versions of major browsers. For the sake of security and providing the best experience to the majority of customers, we do not support less popular browsers that no longer receive security updates.

| Name                    | Versions              |
| ----------------------- | --------------------- |
| Chrome                  | Last 3 major versions |
| Chrome Mobile           | Last 3 major versions |
| Firefox                 | Last 3 major versions |
| Microsoft Edge          | Last 3 major versions |
| Safari OSX              | Last 3 major versions |
| Safari iOS              | Last 3 major versions |
| Android default browser | Last 3 major versions |

Footprint products depend on certain native browser functionality. Polyfilling certain browser features (e.g Promises API) may prevent footprint.js from working normally. If footprint.js isn't functioning as expected, try removing polyfills to determine if they might be causing the issue.

## Footprint.js Library

### How to install footprint.js

1. Install `footprint-js` :

```bash
# With NPM
npm install @onefootprint/footprint-js

# With yarn
yarn add @onefootprint/footprint-js
```

### How to integrate footprint.js

1. Grab the Onboarding Publishable Key, for example, `ob_test_VMooXd04EUlnu3AvMYKjMW`.

2. Import the styles file and embed the Footprint button, passing the Publishable Key and the callback functions:

```html
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <meta name="theme-color" content="#000000" />
    <title>Footprint | Vanilla</title>
    <link
      rel="stylesheet"
      href="https://unpkg.com/@onefootprint/footprint-js@latest/dist/footprint-js.css"
    />
    <script>
      window.onFootprintCompleted = function (validationToken) {
        // TODO: Post the token to your server
        console.log(validationToken);
      };
      window.onFootprintCanceled = function () {
        console.log('canceled');
      };
    </script>
    <style>
      main {
        display: flex;
        flex-direction: column;
        align-items: center;
        justify-content: center;
        height: 100vh;
      }
    </style>
  </head>
  <body>
    <noscript>You need to enable JavaScript to run this app.</noscript>
    <main>
      <div
        id="footprint-button"
        data-public-key="ob_test_WNgSBRR7uxoT8JRDBBflgw"
      ></div>
    </main>
    <script
      src="https://unpkg.com/@onefootprint/footprint-js@latest/dist/footprint-js.umd.js"
      crossorigin
    ></script>
  </body>
</html>
```

What's happening under the hood?

1. Footprint will identify an element with the id `footprint-button`, and will render our button inside it. Using the `public-key`, passed
   via the `data` attribute, it will start the KYC flow using your onboarding configuration.

2. Once the user completes the flow, you'll receive the `validationToken` through the `window.onFootprintCompleted` event, and from there
   you can post it to your backend.

Click [here](https://github.com/onefootprint/examples/tree/master/frontend-vanilla-js) to check out a full example.

### Usage with frameworks

Let's imagine your app was written using [Vue](https://vuejs.org/) (could be any framework or library). To integrate with Footprint,
you'll need to:

1. Render Footprint Button
2. Open Footprint once the button is clicked

This can be achieved easily with our `createButton` and `show` methods:

```vue
<script>
import footprint from '@onefootprint/footprint-js';

export default {
  mounted() {
    const container = document.getElementById('footprint-container');
    const button = footprint.createButton(container);
    button.addEventListener('click', this.handleOpen);
  },

  methods: {
    handleOpen() {
      footprint.show({
        publicKey: 'ob_test_WNgSBRR7uxoT8JRDBBflgw',
      });
    },
  },
};
</script>

<template>
  <div id="footprint-container"></div>
</template>
```

Click [here](https://github.com/onefootprint/examples/tree/master/frontend-vite-vue) to check out a full example.

### Custom button

In fact you can open Footprint from any button or action. To do that, just call `footprint.show` on your event handler.

```vue
<script>
import footprint from '@onefootprint/footprint-js';

export default {
  methods: {
    handleOpen() {
      footprint.show({
        publicKey: 'ob_test_WNgSBRR7uxoT8JRDBBflgw',
      });
    },
  },
};
</script>

<template>
  <button @click="handleOpen">Verify</button>
</template>
```

### Listening to events

Footprint provides some events based on some actions performed by the user. To listen to event, just pass it from the `show` method.

```typescript
footprint.show({
  publicKey: 'ob_test_WNgSBRR7uxoT8JRDBBflgw',
  onCompleted: validationToken => {
    console.log(validationToken);
  },
});
```

| Name        | Description                                                                                                                     |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------- |
| onCompleted | Triggered after completing the onboarding flow. You'll receive a `validationToken`, and from there you can post to your backend |
| onCanceled  | Triggered when the user abandons the flow. This can be triggered when the user clicks on the close button inside our iframe     |

## Footprint-components.js Library

### How to install footprint-components.js

1. Install `footprint-components-js` :

```bash
# With NPM
npm install @onefootprint/footprint-components-js

# With yarn
yarn add @onefootprint/footprint-components-js
```

### How to integrate FootprintSecureForm

1. Perform steps 1-5 from [here](https://docs.onefootprint.com/embedded-components/getting-started#add-footprint-components-js-to-your-app).

2. Import the styles file and embed the Footprint component, passing the Auth token, card alias, callback functions and any other options:

```html
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <meta name="theme-color" content="#000000" />
    <title>Footprint Components | Vanilla</title>
    <link
      rel="stylesheet"
      href="https://unpkg.com/@onefootprint/footprint-components-js@latest/dist/footprint-components-js.css"
    />
    <script>
      window.footprintComponentsCallbacks = {
        onSave: function () {
          // TODO: Decrypt or proxy the saved data from your backend
        },
        onCancel: function () {
          // TODO:
        },
        onClose: function () {
          // TODO:
        },
      };
    </script>
    <style>
      main {
        display: flex;
        flex-direction: column;
        align-items: center;
        justify-content: center;
        height: 100vh;
      }
    </style>
  </head>
  <body>
    <noscript>You need to enable JavaScript to run this app.</noscript>
    <main>
      <div
        id="footprint-component"
        data-kind="secure-form"
        data-props='{"cardAlias":"primary","authToken":"tok_joXzzB0kIVW0fMCB7RWPAHWt8itWdFWpit", variant: 'card'}'
      ></div>
    </main>
    <script
      src="https://unpkg.com/@onefootprint/footprint-components-js@latest/dist/footprint-components-js.umd.js"
      crossorigin
    ></script>
  </body>
</html>
```

What's happening under the hood?

1. Footprint will identify an element with the id `footprint-component`, and will render a secure component based on the data provided.

   - `data-kind` attribute determines the kind of the secure component to render
   - `data-props` are the stringified props objects that will be passed into your component

2. When the user has completed filling the secure collection form on your product, you can decrypt the data collected. More details can be found [here](https://docs.onefootprint.com/vault/apis#decrypt-data-from-a-users-vault)

Click [here](https://github.com/onefootprint/examples/tree/master/frontend-vanilla-js) to check out a full example.

### Usage with frameworks

Let's imagine your app was written using [Vue](https://vuejs.org/) (could be any framework or library). To integrate with Footprint components,
you'll need to tell the integration library the container ID the component should be rendered inside.

This can be achieved easily with our `render` method:

```vue
<script>
import footprintComponent from '@onefootprint/footprint-components-js';

export default {
  mounted() {
    footprintComponent.render({
      kind: 'secure-form',
      containerId: 'footprint-secure-form',
      props: {
        authToken: 'tok_joXzzB0kIVW0fMCB7RWPAHWt8itWdFWpit', // auth token generated using the Secret API Key on step 5
        cardAlias: 'primary', // this should match the cardAliases used while generating the auth token
        onSave: handleSave,
      },
    });
  },
  methods: {
    handleSave() {
      // TODO: Decrypt or proxy the saved data from your backend
    },
  },
};
</script>

<template>
  <div id="footprint-secure-form"></div>
</template>
```

If you are adding multiple Footprint components into your app, it is important to choose unique container IDs for each of them.

Click [here](https://github.com/onefootprint/examples/tree/master/frontend-vite-vue) to check out a full example.

### Listening to SecureForm events

Our Secure Form component provides some events based on actions performed by the user. To listen to event, pass callbacks into the props attribute.

```typescript
footprintComponent.render({
  kind: 'secure-form'
  props: {
    authToken,
    cardAlias,
    title: 'Add a New Card',
    type: 'cardAndName',
    variant: 'drawer',
    onSave: handleSave,
    onClose: handleClose,
    onCancel: handleCancel
  },
  containerId: 'footprint-secure-form',
});
```

| Name     | Description                                                                                                                        |
| -------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| onSave   | Triggered after user completes the form and data is saved to the user vault.                                                       |
| onCancel | Triggered when the user abandons the flow. This can be triggered when the user clicks on close or cancel buttons inside our iframe |
| onClose  | Triggered when the user abandons the flow. This can be triggered when the user clicks on close or cancel buttons inside our iframe |
