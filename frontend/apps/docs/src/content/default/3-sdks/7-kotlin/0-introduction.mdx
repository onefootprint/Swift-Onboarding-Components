# FootprintAndroid package overview

You can use our `FootprintAndroid` package for adding Footprint-powered onboarding flows to your Kotlin or Java application. 

## Minimum version requirements

Our onboarding flows require a `compileSdk` version of 34 or above in your app's gradle file.

If you're using older versions, please consider upgrading to ensure compatibility with `FootprintAndroid`. Failure to meet these requirements could lead to unexpected issues or the package not working at all.

## Permissions

In order to use the `FootprintAndroid` package your manifest must include `INTERNET` permission:

```xml
<uses-permission android:name="android.permission.INTERNET" />
```

## Installation

In order to use our SDK, you will need to add it use Jitpack, which is a package repository for JVM and Android projects that builds Git projects on demand and provides you with ready-to-use artifacts (jar, aar).

If you are using Gradle for your build system, add the following to your `settings.gradle` file: 

```kotlin
pluginManagement {
    repositories {
        google()
        mavenCentral()
        gradlePluginPortal()
        maven("https://jitpack.io") // Add this line
    }
}

dependencyResolutionManagement {
    repositories {
        google()
        mavenCentral()
        maven("https://jitpack.io") // Add this line
    }
}
```

For instructions on how to install with different build systems (e.g. maven, sbt, leiningen), see [here](https://jitpack.io/#onefootprint/footprint-android)

## Available components

There are two ways to utilize FootprintAndroid:

### Hosted

The Hosted integration is the solution that allows the Footprint's flow to be executed in a secure Chrome Custom Tab inside your app. It requires a very minimal setup and is the easiest way to get started.

You can find more information [here](/articles/sdks/android-hosted).

### Onboarding components

The Onboarding components integration allows you to embed the Footprint's flow directly into your app, giving you full control over the design and layout.

You can find more information [here](/articles/sdks/android-onboarding-components).
