## Installation

In your app's `build.gradle.kts` file, add the following line in the dependencies block:

```kotlin
dependencies {
    implementation 'com.github.onefootprint:footprint-android:<VERSION>'
}
```

You should replace `<VERSION>` with the version number, like `1.0.0`. We recommend always using the latest available version published. You can find the published versions [here](https://jitpack.io/#onefootprint/footprint-android).


## Getting started

This guide explains how to integrate the Footprint onboarding flow for KYC (Know Your Customer) and KYB (Know Your Business) into your React Native applications. This is our Hosted integration, meaning that Footprint's flow will be executed in a Webview, and hence, it's our most simple and quick integration.

1. Start by getting your Playbook public key, for instance, `pb_test_VMooXd04EUlnu3AvMYKjMW` from your [Footprint Dashboard](https://dashboard.onefootprint.com/playbooks).

2. Make sure to import `FootprintAndroid` from `import com.footprint.android.FootprintAndroid` to instantiate the flow.

3. Next, call the `FootprintAndroid`'s `init` method to trigger the verification flow. You will need to pass a configuration into `init`. You should pass your public key and destination activity name, which is the activity on your app that you want the app to be on after the verification flow has completed. The name must contain the package name as well. For example, if your activity’s class is named `MainActivity` and it is in the package `com.example.package`, the destination activity name will be `com.example.package.MainActivity`. You can also add callback for when the onboarding flow is completed or canceled.

```kotlin
package com.example.footprint_verification

import androidx.appcompat.app.AppCompatActivity
import android.os.Bundle
import android.util.Log
import android.widget.Button
import com.footprint.android.FootprintAndroid
import com.footprint.android.FootprintConfig

class MainActivity : AppCompatActivity() {
    private lateinit var verificationButton: Button

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
        verificationButton = findViewById(R.id.verify_button)
        verificationButton.setOnClickListener {
            val config = FootprintConfig(
                redirectActivityName = "com.example.footprint_verification.MainActivity",
                publicKey = "pb_test_aSzwnZecnXS4faoyhxrocW",
                onComplete = {token: String ->
                    Log.d("VerificationResult", "The flow has completed. The validation token is $token")
                },
                onCancel = {
                    Log.d("VerificationResult", "The flow was canceled")
                },
                onError = {
                    Log.d("Footprint error", it)
                }
            )
            FootprintAndroid.init(
                this@MainActivity,
                config = config
            )
        }
    }
}

```

Instead of a `publicKey`, you can also use an `authToken` that you generate for a specific playbook and user. You can find more instructions on how to do this [here](/articles/integrate/user-specific-sessions).

For a complete example, click [here](https://github.com/onefootprint/examples/tree/master/idv/mobile-android-ui).

### Bootstrapping data

Utilize the `bootstrapData` option to pre-fill any available data and bypass completed pages for your users or businesses. You can click [here](/articles/integrate/bootstrap-data) to find out more about the formatting and validation requirements we have for these inputs.

```kotlin
package com.example.footprint_verification

import androidx.appcompat.app.AppCompatActivity
import android.os.Bundle
import android.util.Log
import android.widget.Button
import com.footprint.android.FootprintAndroid
import com.footprint.android.FootprintConfig
import com.footprint.android.FootprintBootstrapData

class MainActivity : AppCompatActivity() {
    private lateinit var verificationButton: Button

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
        verificationButton = findViewById(R.id.verify_button)
        verificationButton.setOnClickListener {
            val bootstrapData = FootprintBootstrapData(
                email = "example@gmail.com",
                phoneNumber = "+15555550100",
                firstName = "Piip",
                lastName = "Foot",
                dob = "01/01/1996",
                addressLine1 = "123 Main St",
                addressLine2 = "Unit 123",
                city = "Huntington Beach",
                state = "CA",
                country = "US",
                zip = "12345",
                ssn9 = "343434344",
                ssn4 = "1234",
                nationality = "US",
                usLegalStatus = "citizen",
                citizenships = listOf("US", "TR"),
                visaKind = "f1",
                visaExpirationDate = "05/12/2024"
            )
            val config = FootprintConfig(
                redirectActivityName = "com.example.footprint_verification.MainActivity",
                publicKey = "pb_test_aSzwnZecnXS4faoyhxrocW",
                bootstrapData = bootstrapData,
                onComplete = {token: String ->
                    Log.d("VerificationResult", "The flow has completed. The validation token is $token")
                }
            )
            FootprintAndroid.init(
                this@MainActivity,
                config = config
            )
        }
    }
}
```

### Customizing the appearance

Take advantage of the `appearance` option to extend the default styles. You can utilize the same variables and rules as in the [web integration](/articles/integrate/customization). You will need to pass values that are valid CSS styles.

```kotlin
package com.example.footprint_verification

import androidx.appcompat.app.AppCompatActivity
import android.os.Bundle
import android.util.Log
import android.widget.Button
import com.footprint.android.FootprintAndroid
import com.footprint.android.FootprintAppearance
import com.footprint.android.FootprintAppearanceRules
import com.footprint.android.FootprintAppearanceVariables
import com.footprint.android.FootprintConfig

class MainActivity : AppCompatActivity() {
    private lateinit var verificationButton: Button

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
        verificationButton = findViewById(R.id.verify_button)
        verificationButton.setOnClickListener {
            val config = FootprintConfig(
                redirectActivityName = "com.example.footprint_verification.MainActivity",
                publicKey = "pb_test_aSzwnZecnXS4faoyhxrocW",
                appearance = FootprintAppearance(
                    rules = FootprintAppearanceRules(button = mapOf("transition" to "all .2s linear")),
                    variables = FootprintAppearanceVariables(borderRadius = "10px", buttonPrimaryBg = "#0C6948")
                ),
                onComplete = {token: String ->
                    Log.d("VerificationResult", "The flow has completed. The validation token is $token")
                }
            )
            FootprintAndroid.init(
                this@MainActivity,
                config = config
            )
        }
    }
}

```

### Showing your company logo

You can also add your company logo at the top of the modal by a passing boolean field `showLogo`.

```kotlin
package com.example.footprint_verification

import androidx.appcompat.app.AppCompatActivity
import android.os.Bundle
import android.util.Log
import android.widget.Button
import com.footprint.android.FootprintAndroid
import com.footprint.android.FootprintConfig
import com.footprint.android.FootprintOptions

class MainActivity : AppCompatActivity() {
    private lateinit var verificationButton: Button

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
        verificationButton = findViewById(R.id.verify_button)
        verificationButton.setOnClickListener {
            val config = FootprintConfig(
                redirectActivityName = "com.example.footprint_verification.MainActivity",
                publicKey = "pb_test_aSzwnZecnXS4faoyhxrocW",
                options = FootprintOptions(showLogo = true),
                onComplete = {token: String ->
                    Log.d("VerificationResult", "The flow has completed. The validation token is $token")
                }
            )
            FootprintAndroid.init(
                this@MainActivity,
                config = config
            )
        }
    }
}
```

You can configure your logo from your [Footprint dashboard](https://dashboard.onefootprint.com/settings).
If no logo is available, we will use a placeholder with your company name's initial
letter.

### Setting the locale

Footprint also supports localization settings, you can use the `l10n` (localization) option and specify the desired locale.

For example, if your audience is in Mexico, you can set the locale to "es-MX" (Spanish - Mexico), Your locale helps Footprint format dates and numbers and adapt to cultural conventions.

```kotlin
package com.example.footprint_verification

import androidx.appcompat.app.AppCompatActivity
import android.os.Bundle
import android.util.Log
import android.widget.Button
import com.footprint.android.FootprintAndroid
import com.footprint.android.FootprintConfig
import com.footprint.android.FootprintL10n
import com.footprint.android.FootprintSupportedLocale

class MainActivity : AppCompatActivity() {
    private lateinit var verificationButton: Button

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)
        verificationButton = findViewById(R.id.verify_button)
        verificationButton.setOnClickListener {
            val config = FootprintConfig(
                redirectActivityName = "com.example.footprint_verification.MainActivity",
                publicKey = "pb_test_aSzwnZecnXS4faoyhxrocW",
                l10n = FootprintL10n(locale = FootprintSupportedLocale.ES_MX),
                onComplete = {token: String ->
                    Log.d("VerificationResult", "The flow has completed. The validation token is $token")
                }
            )
            FootprintAndroid.init(
                this@MainActivity,
                config = config
            )
        }
    }
}
```

## All configuration fields

| Variable               | Description                                                                                                                                                                                                                                                                                                                                          |
| ---------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `redirectActivityName` | Required. The activity on your app that you want the app to be on after the verification flow has completed. The name must contain the package name as well. For example, if your activity’s class is named `MainActivity` and it is in the package `com.example.package` , the destination activity name will be `com.example.package.MainActivity` |
| `publicKey`            | Optional. Onboarding configuration public key.                                                                                                                                                                                                                                                                                                       |
| `authToken`            | Optional. A valid string authentication token generated using the Secret API Key, Footprint User ID and fields to collect. One of publicKey or authToken must be provided. More info [here](/articles/integrate/user-specific-sessions).                                                                                                             |
| `bootstrapData`        | Optional. An object that contains previously collected data that can bootstrap the onboarding flow. More information [here](/articles/integrate/bootstrap-data).                                                                                                                                                                                     |
| `options`              | Optional. An options object that customizes the flow, can accept `showLogo` and `showCompletionPage` entries. More info [here](/articles/integrate/customization#available-options)                                                                                                                                                                  |
| `appearance`           | Optional. A `FootprintAppearance` object that customizes the look of your integration                                                                                                                                                                                                                                                                |
| `l10n`                 | Optional. Specifies the desired localization. More information [here](/articles/integrate/customization#localization-configuration).                                                                                                                                                                                                                 |
| `onComplete`           | Triggered after the user completes the onboarding flow. You'll receive a `validationToken` that your backend can exchange with Footprint to see the fp_id, the login method used, and the KYC status.                                                                                                                                                |
| `onCancel`             | Optional. A function that is called when the onboarding flow is canceled by the user.                                                                                                                                                                                                                                                                |
| `onError`              | Optional. A function that is called there was an unrecoverable error while initializing the onboarding flow. It takes in an error string argument with more details.                                                                                                                                                                                 |
