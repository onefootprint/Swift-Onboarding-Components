## Getting started

`<FootprintForm />` is our PCI-compliant form to collect user data for specific fields, for example credit card payment information.

1. **Before starting**:

- We recommend you reading our [quick-start guide](/embedded-components/getting-started), which explains a bit more about this product.

2. **Obtain an `authToken`**:

- Perform steps 1-5 from [here](/embedded-components/getting-started#add-footprint-js-to-your-app).

3. **Initialize the Footprint Flow**:

   - Import the styles file and embed the Footprint component, passing the `authToken`, callback functions and any other options:

```typescript
import '@onefootprint/footprint-js/dist/footprint-js.css';

import { FootprintForm } from '@onefootprint/footprint-react';

const Page = () => {
  const handleSave = () => {
    // TODO: Decrypt or proxy the saved data from your backend
  };

  const handleCancel = () => {
    // TODO:
  };

  const handleClose = () => {
    // TODO:
  };

  return (
    <main>
      <FootprintForm
        authToken={secretAuthToken}
        variant="modal"
        title="Payment information"
        onSave={handleSave}
        onClose={handleClose}
        onCancel={handleClose}
      />
    </main>
  );
};
```

4. **Handle Completion:**

- When the user has completed filling the secure collection form on your product, you can decrypt the data collected. More details can be found [here](/vault/apis#decrypt-data-from-a-users-vault).

## Listening to events

Footprint provides some events based on some actions performed by the user.

```typescript
import '@onefootprint/footprint-js/dist/footprint-js.css';

import { FootprintVerifyButton } from '@onefootprint/footprint-react';

const Page = () => {
  const handleComplete = (validationToken: string) => {
    console.log(validationToken);
  };

  const handleError = (error: unknown) => {
    console.log(error);
  };

  const handleCancel = () => {
    console.log('User canceled the flow');
  };

  const handleClose = () => {
    console.log('User closed the flow');
  };

  return (
    <main>
      <FootprintVerifyButton
        authToken={secretAuthToken}
        onComplete={handleComplete}
        onError={handleError}
        onCancel={handleCancel}
        onClose={handleClose}
      />
    </main>
  );
};

export default Page;
```

## Setting a custom appearance

You can customize the appearance of the onboarding flow by passing an `appearance` object to the `init` method.

```typescript
import '@onefootprint/footprint-js/dist/footprint-js.css';

import { FootprintVerifyButton } from '@onefootprint/footprint-react';

const Page = () => {
  return (
    <main>
      <FootprintVerifyButton
        authToken={secretAuthToken}
        appearance={{
          variables: {
            borderRadius: '8px',
            colorSuccess: '#10b981',
            colorError: '#F87171',
            buttonPrimaryBg: '#5550e9',
          }
        }}
      />
    </main>
  );
};
```

For more information, including a list of available variables, check out the [customization guide](/integrate/customization).

### Available props

| Variable     | Description                                                                                                                                                                                                                                                           |
| ------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --- |
| `title`      | Optional. A form header.                                                                                                                                                                                                                                              |
| `authToken`  | Optional. A valid string authentication token generated using the Secret API Key, Footprint User ID and fields to collect. More info [here](/integrate/user-specific-sessions).                                                                                       |
| `onComplete` | Triggered after user completes the form and data is saved to the user vault.                                                                                                                                                                                          |
| `onError`    | Optional. A function that is called there was an unrecoverable error while initializing the onboarding flow. It takes in an error string argument with more details.                                                                                                  |
| `onCancel`   | Triggered when the user abandons the flow. This can be triggered when the user clicks on close or cancel buttons inside our iframe                                                                                                                                    |
| `onClose`    | Triggered when the user abandons the flow. This can be triggered when the user clicks on close or cancel buttons inside our iframe                                                                                                                                    |
| `options`    | Optional. An options object that customizes the flow, can accept `hideButtons`, `hideCancelButton` and `hideFootprintLogo` boolean entries.                                                                                                                           |
| `appearance` | Optional. A `FootprintAppearance` object that customizes the look of your integration                                                                                                                                                                                 |
| `variant`    | Optional. One of `modal`, `inline` or `drawer`.                                                                                                                                                                                                                       |
| `getRef`     | Optional. An async callback that takes one argument and returns a promise. The `ref` object contains a method `save()` that can be called to save the form data (`ref.save()`). Promise is resolved or rejected with a string error when saving to vault is complete. |     |
