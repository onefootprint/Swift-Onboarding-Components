# Available Components

Our SDKs support a variety of components.

| Component                   | Description                                                                                                       |
| --------------------------- | ----------------------------------------------------------------------------------------------------------------- |
| `<FootprintVerifyButton />` | A button that launches the footprint onboarding flow in a modal or drawer.                                        |
| `<FootprintForm />`         | A PCI-compliant form to collect user data for specific fields. It can be rendered inline, in a modal or a drawer. |
| `<FootprintAuthButton />`   | A button that launches the footprint authentication flow in a modal or drawer.                                    |

You will find more information on each of these below:

## Installation

1. Install `footprint-js` and `footprint-react`:

```bash
# With NPM
npm install @onefootprint/footprint-js
npm install @onefootprint/footprint-react

# With yarn
yarn add @onefootprint/footprint-js
yarn add @onefootprint/footprint-react
```

<inline-alert variant='info'>Make sure your application's permissions and content security policies are configured correctly to allow the Footprint integration. More info can be found [here](/integrate/content-security-policy).</inline-alert>

## FootprintVerifyButton Integration

`<FootprintVerifyButton />` is a button that launches our onboarding flow when clicked. The flow can be rendered in a modal or drawer.

1. Grab the Onboarding Publishable Key, for example `pb_test_VMooXd04EUlnu3AvMYKjMW`.

2. Import the styles file and embed the Footprint button, passing the Publishable Key and the callback functions:

```typescript
import '@onefootprint/footprint-js/dist/footprint-js.css';
import { FootprintVerifyButton } from '@onefootprint/footprint-react';

const Page = () => {
  const handleCancel = () => {
    console.log('user canceled');
  };

  const handleComplete = (validationToken: string) => {
    console.log(validationToken);
  };

  return (
    <main>
      <FootprintVerifyButton
        publicKey="pb_test_VMooXd04EUlnu3AvMYKjMW"
        onComplete={handleComplete}
        onCancel={handleCancel}
      />
    </main>
  );
};
```

Click [here](https://github.com/onefootprint/examples/tree/master/idv/frontend-react-cra) to check out a full example. We also have some examples using [Next.js](https://github.com/onefootprint/examples/tree/master/idv/frontend-react-typescript-next)

### Setting user data

Depending on your onboarding configuration, Footprint might request customer's phone number and email. In some circustances,
you might have this data already available, so Footprint doesn't need to request this data once more. To pass user data to Footprint's flow,
use the prop `userData`:

```typescript
import '@onefootprint/footprint-js/dist/footprint-js.css';

import { FootprintVerifyButton } from '@onefootprint/footprint-react';

const Page = () => {
  return (
    <main>
      <FootprintVerifyButton
        publicKey="pb_test_VMooXd04EUlnu3AvMYKjMW"
        userData={{
          'id.email': 'jane.doe@acme.com',
          'id.phone_number': '+12025550179',
        }}
      />
    </main>
  );
};
```

Click [here](https://github.com/onefootprint/examples/tree/master/idv/frontend-react-user-data) to check out a full example and [here](https://docs.onefootprint.com/integrate/user-data) for more details about user data.

### Available FootprintVerifyButton Props

| Variable        | Description                                                                                                                                                                                             |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `label`         | Optional. A string that will be displayed on the button                                                                                                                                                 |
| `publicKey`     | Optional. Onboarding configuration public key that will be used to launch the onboarding modal when button is clicked.                                                                                  |
| `authToken`     | Optional. A valid string authentication token generated using the Secret API Key, Footprint User ID and fields to collect.                                                                              |
| `dialogVariant` | Optional. One of `modal` or `drawer`. Specifies how the onboarding dialog should be displayed when launched.                                                                                            |
| `onClick`       | Optional. A function that is called when the button is clicked.                                                                                                                                         |
| `onComplete`    | Optional. A function that is called when the onboarding flow is completed by the user.                                                                                                                  |
| `onCancel`      | Optional. A function that is called when the onboarding flow is canceled by the user.                                                                                                                   |
| `onClose`       | Optional. A function that is called when the onboarding flow is closed by the user.                                                                                                                     |
| `userData`      | Optional. An object that contains previously collected user data that can bootstrap the onboarding flow. More information [here](https://www.npmjs.com/package/@onefootprint/footprint-react).          |
| `options`       | Optional. An options object that customizes the flow, can accept `showLogo` and `showCompletionPage` entries. More info [here](https://docs.onefootprint.com/integrate/customization#available-options) |
| `appearance`    | Optional. A `FootprintAppearance` object that customizes the look of your integration                                                                                                                   |
| `testID`        | Optional. A test string to use for UI tests.                                                                                                                                                            |
| `ref`           | Optional. Access to the DOM button node managed by React.                                                                                                                                               |
| `l10n`          | Optional. Specifies the desired locale. More information [here](https://docs.onefootprint.com/integrate/customization#localization-configuration).                                                      |

## FootprintForm Integration

`<FootprintForm />` is our PCI-compliant form to collect user data for specific fields, for example credit card payment information.

1. Perform steps 1-5 from [here](https://docs.onefootprint.com/embedded-components/getting-started#add-footprint-js-to-your-app).

2. Import the styles file and embed the Footprint component, passing the Auth token, callback functions and any other options:

```typescript
import '@onefootprint/footprint-js/dist/footprint-js.css';

import { FootprintForm } from '@onefootprint/footprint-react';

const Page = () => {
  const handleSave = () => {
    // TODO: Decrypt or proxy the saved data from your backend
  };

  const handleCancel = () => {
    // TODO:
  };

  const handleClose = () => {
    // TODO:
  };

  return (
    <main>
      <FootprintForm
        authToken={secretAuthToken}
        variant="modal"
        title="Payment information"
        onSave={handleSave}
        onClose={handleClose}
        onCancel={handleClose}
      />
    </main>
  );
};
```

Click [here](https://github.com/onefootprint/examples/tree/master/form/frontend-react-cra) to check out a full example. We also have some examples using [Next.js](https://github.com/onefootprint/examples/tree/master/auth/frontend-react-typescript-next)

### Available FootprintForm Props

| Variable     | Description                                                                                                                                                                                                                                                           |
| ------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --- |
| `title`      | Optional. A form header.                                                                                                                                                                                                                                              |
| `authToken`  | Optional. A valid string authentication token generated using the Secret API Key, Footprint User ID and fields to collect.                                                                                                                                            |
| `onComplete` | Triggered after user completes the form and data is saved to the user vault.                                                                                                                                                                                          |
| `onCancel`   | Triggered when the user abandons the flow. This can be triggered when the user clicks on close or cancel buttons inside our iframe                                                                                                                                    |
| `onClose`    | Triggered when the user abandons the flow. This can be triggered when the user clicks on close or cancel buttons inside our iframe                                                                                                                                    |
| `options`    | Optional. An options object that customizes the flow, can accept `hideButtons`, `hideCancelButton` and `hideFootprintLogo` boolean entries.                                                                                                                           |
| `appearance` | Optional. A `FootprintAppearance` object that customizes the look of your integration                                                                                                                                                                                 |
| `variant`    | Optional. One of `modal`, `inline` or `drawer`.                                                                                                                                                                                                                       |
| `getRef`     | Optional. An async callback that takes one argument and returns a promise. The `ref` object contains a method `save()` that can be called to save the form data (`ref.save()`). Promise is resolved or rejected with a string error when saving to vault is complete. |     |

### Available FootprintAuthButton Props

| Variable     | Description                                                                                                                                                                          |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `publicKey`  | Optional. Onboarding configuration public key.                                                                                                                                       |

| `l10n`       | Optional. Specifies the desired locale. More information [here](https://docs.onefootprint.com/integrate/customization#localization-configuration).                                   |
