## Footprint.js Library

### Installation

1. Install `footprint-js` and `footprint-react`:

```bash
# With NPM
npm install @onefootprint/footprint-js
npm install @onefootprint/footprint-react

# With yarn
yarn add @onefootprint/footprint-js
yarn add @onefootprint/footprint-react
```

### Integration

1. Grab the Onboarding Publishable Key, for example `ob_test_VMooXd04EUlnu3AvMYKjMW`.

2. Import the styles file and embed the Footprint button, passing the Publishable Key and the callback functions:

```typescript
import '@onefootprint/footprint-js/dist/footprint-js.css';

import { FootprintButton } from '@onefootprint/footprint-react';

const Page = () => {
  const handleCompleted = (validationToken: string) => {
    console.log(validationToken);
  };

  const handleCanceled = () => {
    console.log('user canceled');
  };

  return (
    <main>
      <FootprintButton
        publicKey="ob_test_VMooXd04EUlnu3AvMYKjMW"
        onCompleted={handleCompleted}
        onCanceled={handleCanceled}
      />
    </main>
  );
};
```

Click [here](https://github.com/onefootprint/examples/tree/master/frontend-react-cra) to check out a full example. We also have some examples using [Next.js](https://github.com/onefootprint/examples/tree/master/frontend-react-typescript-next)

### Customizing appearance

You can customize the appearance of our flow using the `appearance` prop:

```typescript
import '@onefootprint/footprint-js/dist/footprint-js.css';

import { FootprintButton } from '@onefootprint/footprint-react';

const Page = () => {
  return (
    <main>
      <FootprintButton
        publicKey="ob_test_VMooXd04EUlnu3AvMYKjMW"
        appearance={{
          borderRadius: 0,
          inputBg: 'red',
        }}
      />
    </main>
  );
};
```

Click [here](https://github.com/onefootprint/examples/tree/master/frontend-react-customized) to check out a full example and [here](https://docs.onefootprint.com/integrate/customization) for more details about customization.

### Setting user data

Depending on your onboarding configuration, Footprint might request customer's phone number and email. In some circustances,
you might have this data already available, so Footprint doesn't need to request this data once more. To pass user data to Footprint's flow,
use the prop `userData`:

```typescript
import '@onefootprint/footprint-js/dist/footprint-js.css';

import { FootprintButton } from '@onefootprint/footprint-react';

const Page = () => {
  return (
    <main>
      <FootprintButton
        publicKey="ob_test_VMooXd04EUlnu3AvMYKjMW"
        userData={{
          email: 'jane.doe@acme.com',
          phoneNumber: '+12025550179',
        }}
      />
    </main>
  );
};
```

Click [here](https://github.com/onefootprint/examples/tree/master/frontend-react-user-data) to check out a full example and [here](https://docs.onefootprint.com/integrate/user-data) for more details about user data.

## Footprint-components.js Library

### Installation

1. Install `footprint-components-js` and `footprint-components-react`:

```bash
# With NPM
npm install @onefootprint/footprint-components-js
npm install @onefootprint/footprint-components-react

# With yarn
yarn add @onefootprint/footprint-components-js
yarn add @onefootprint/footprint-components-react
```

### Integration

1. Perform steps 1-5 from [here](https://docs.onefootprint.com/embedded-components/getting-started#add-footprint-components-js-to-your-app).

2. Import the styles file and embed the Footprint component, passing the Auth token, card alias, callback functions and any other options:

```typescript
import '@onefootprint/footprint-components-js/dist/footprint-components-js.css';

import { SecureFormType } from '@onefootprint/footprint-components-js';
import { FootprintSecureForm } from '@onefootprint/footprint-components-react';

const Page = () => {
  const handleSave = () => {
    // TODO: Decrypt or proxy the saved data from your backend
  };

  const handleCancel = () => {
    // TODO:
  };

  const handleClose = () => {
    // TODO:
  };

  return (
    <main>
      <FootprintSecureForm
        authToken={secretAuthToken}
        cardAlias="primary"
        type={SecureFormType.cardAndZip}
        variant="modal"
        title="Payment information"
        onSave={handleSave}
        onClose={handleClose}
        onCancel={handleClose}
      />
    </main>
  );
};
```

Click [here](https://github.com/onefootprint/examples/tree/master/frontend-react-cra) to check out a full example. We also have some examples using [Next.js](https://github.com/onefootprint/examples/tree/master/frontend-react-typescript-next)

### Customizing appearance

You can customize the appearance of our embedded components using the `appearance` prop:

```typescript
import '@onefootprint/footprint-components-js/dist/footprint-components-js.css';

import {
  FootprintAppearance,
  SecureFormType,
} from '@onefootprint/footprint-components-js';
import { FootprintSecureForm } from '@onefootprint/footprint-components-react';

const Page = () => {
  const appearance: FootprintAppearance = {
    borderRadius: 0,
    inputBg: 'red',
  };

  return (
    <main>
      <FootprintSecureForm
        authToken={secretAuthToken}
        cardAlias="primary"
        type={SecureFormType.cardAndZip}
        appearance={appearance}
      />
    </main>
  );
};
```

Click [here](https://github.com/onefootprint/examples/tree/master/frontend-react-customized) to check out a full example and [here](https://docs.onefootprint.com/integrate/customization) for more details about customization.

### Available SecureForm Props

| Variable     | Description                                                                                                                                                              |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `authToken`  | Required. A valid string authentication token generated using the Secret API Key, Footprint User ID and fields to collect                                                |
| `cardAlias`  | Required. A string that is a unique alias for the card information                                                                                                       |
| `title`      | Optional. A string that will be displayed at the top of the field                                                                                                        |
| `type`       | Optional. Of type SecureFormType, one of `cardOnly`, `cardAndName`, `cardAndNameAndAddress`, `cardAndZip`. Specifies which fields to collect. Defaults to `cardAndName`. |
| `variant`    | Optional. One of `modal`, `drawer` or `card`. Specifies whether the form will be rendered in-line or in a modal.                                                         |
| `appearance` | Optional. A stringified `FootprintAppearance` object that can be imported from to customize the look of your integration                                                 |
