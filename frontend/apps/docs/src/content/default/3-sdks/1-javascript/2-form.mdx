## Getting started

This guide explains how to integrate the Footprint Form component into your JavaScript/TypeScript applications.

1. **Before starting**:

- We recommend you reading our [quick-start guide](/articles/embedded-components/getting-started), which explains a bit more about this product.

2. **Obtain an `authToken`**:

- Perform steps 1-5 from [here](/articles/embedded-components/getting-started#add-footprint-js-to-your-app).

3. **Initialize the Footprint Flow**:

   - Import the styles file and embed the Footprint component, passing the `authToken`, callback functions and any other options:

```javascript
import "@onefootprint/footprint-js/dist/footprint-js.css";
import footprint from "@onefootprint/footprint-js";

const renderForm = () => {
  const component = footprint.init({
    kind: "form",
    variant: "inline",
    containerId: "myContainerId", // unique ID for the div that the form should be rendered in
    authToken: "utok_123",
    onComplete: () => {
      // TODO:
    },
  });
  component.render();
};
```

4. **Handle Completion:**

- When the user has completed filling the secure collection form on your product, you can decrypt the data collected. More details can be found [here](/articles/vault/apis#decrypt-data-from-a-users-vault).

Click [here](https://github.com/onefootprint/examples/tree/master/idv/frontend-vite-vanilla) to check out a full example.

## Listening to events

Our Form component provides some events based on actions performed by the user. To listen to event, pass callbacks into the props attribute.

```typescript
const component = footprint.init({
  kind: 'form'
  authToken,
  title: 'Add a New Card',
  variant: 'inline',
  containerId: 'my-form',
  onSave: () => {
    console.log('Form saved');
  },
  onCancel: () => {
    console.log('User canceled the flow');
  },
    onClose: () => {
    console.log('User closed the flow');
  },
});

component.render();
```

## Setting a custom appearance

You can customize the appearance of the onboarding flow by passing an `appearance` object to the `init` method.

```typescript
const component = footprint.init({
  kind: "verify",
  publicKey: "pb_test_WNgSBRR7uxoT8JRDBBflgw",
  onComplete: (validationToken) => {
    console.log(validationToken);
  },
  appearance: {
    variables: {
      borderRadius: "8px",
      colorSuccess: "#10b981",
      colorError: "#F87171",
      buttonPrimaryBg: "#5550e9",
    },
  },
});
component.render();
```

For more information, including a list of available variables, check out the [customization guide](/articles/integrate/customization).

## Available Props

| Variable      | Description                                                                                                                                                                                                                                                           |
| ------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --- |
| `title`       | Optional. A form header.                                                                                                                                                                                                                                              |
| `authToken`   | Optional. A valid string authentication token generated using the Secret API Key, Footprint User ID and fields to collect. More info [here](/articles/integrate/user-specific-sessions).                                                                              |
| `onComplete`  | Triggered after user completes the form and data is saved to the user vault.                                                                                                                                                                                          |
| `onError`     | Optional. A function that is called there was an unrecoverable error while initializing the onboarding flow. It takes in an error string argument with more details.                                                                                                  |
| `onCancel`    | Triggered when the user abandons the flow. This can be triggered when the user clicks on close or cancel buttons inside our iframe                                                                                                                                    |
| `onClose`     | Triggered when the user abandons the flow. This can be triggered when the user clicks on close or cancel buttons inside our iframe                                                                                                                                    |
| `options`     | Optional. An options object that customizes the flow, can accept `hideButtons`, `hideCancelButton` and `hideFootprintLogo` boolean entries.                                                                                                                           |
| `appearance`  | Optional. A `FootprintAppearance` object that customizes the look of your integration                                                                                                                                                                                 |
| `variant`     | Optional. One of `modal`, `inline` or `drawer`. If `inline`, requires a `containerId`. Specifies how the form will be rendered.                                                                                                                                       |
| `containerId` | Required if `variant` is `inline`. The ID for the HTML element that the form will be rendered in.                                                                                                                                                                     |
| `getRef`      | Optional. An async callback that takes one argument and returns a promise. The `ref` object contains a method `save()` that can be called to save the form data (`ref.save()`). Promise is resolved or rejected with a string error when saving to vault is complete. |     |
