## Getting started

This guide explains how to integrate the Footprint onboarding flow for KYC (Know Your Customer) and KYB (Know Your Business) into your React Native applications. This is our _Hosted_ integration, meaning that Footprint's flow will
be executed in a Webview, and hence, it's our most simple and quick integration.

1. **Access Your Dashboard**:

   - Go to the [playbooks page](https://dashboard.onefootprint.com/playbooks) on your dashboard.
   - Use an existing playbook or create a new one.
   - Obtain the Publishable Key, e.g., `pb_test_LRfe5D3dZnWSf5vOGSeCFk`.

2. **Initialize the Footprint Flow**:
   - Trigger Footprint, for example, when a button is pressed. Then, pass the `publicKey` and `onComplete` callback to the `init` method.

```typescript
import footprint from '@onefootprint/footprint-react-native';

const App = () => {
  const launchFootprint = () => {
    const fp = footprint.init({
      publicKey: 'pb_test_LRfe5D3dZnWSf5vOGSeCFk',
      redirectUrl: Platform.OS === 'android' ? 'footprint://callback/ : `footprint://`,
      onComplete: validationToken => {
        console.log(validationToken);
      },
    });

    fp.render();
  };

  return (
    <View>
      <Button title="Verify" onPress={launchFootprint} />
    </View>
  );
}
```

3. **Handle Completion:**
   - Once the user completes the flow, you'll receive the validationToken through the `onComplete` callback. Post this token to your backend for further processing.

<br />

Click [here](https://github.com/onefootprint/examples/tree/master/idv/mobile-react-native) to check out a full example.

## Listening to events

Footprint provides some events based on some actions performed by the user. To listen to event, just pass it from the `init` method.

```typescript
const fp = footprint.init({
  publicKey: "pb_test_LRfe5D3dZnWSf5vOGSeCFk",
  redirectUrl: Platform.OS === 'android' ? 'footprint://callback/ : `footprint://`,
  onComplete: (validationToken) => {
    console.log(validationToken);
  },
  onError: (error) => {
    console.log(error);
  },
  onAuth: (validationToken) => {
    console.log(validationToken);
  },
  onCancel: () => {
    console.log("User canceled the flow");
  },
  onClose: () => {
    console.log("User closed the flow");
  },
});

fp.render();
```

### Bootstraping data

Utilize the `bootstrapData` option to pre-fill any available data and bypass completed pages for your users. For complete details, click [here](/integrate/bootstrap-data).

```typescript
import footprint from '@onefootprint/footprint-react-native';

import { View, Button } from 'react-native';

const Screen = () => {
  const handlePress = () => {
    const fp = footprint.init({
      publicKey: 'pb_test_VMooXd04EUlnu3AvMYKjMW',
      redirectUrl: Platform.OS === 'android' ? 'footprint://callback/ : `footprint://`,
      bootstrapData: {
        'id.email': 'jane.doe@acme.com',
        'id.phone_number': '+12025550179',
        'id.first_name': 'Jane',
        'id.last_name': 'Doe',
        'id.dob': '01/01/2001',
        'id.ssn9': '123456789',
        'id.ssn4': '1234',
        'id.address_line1': '123 Apple St.',
        'id.address_line2': 'APT 123',
        'id.city': 'Boston',
        'id.state': 'MA',
        'id.country': 'US',
        'id.zip': '02117',
      },
    });
    fp.render();
  };

  return (
    <View>
      <Button title="Verify identity" onPress={handlePress} />
    </View>
  );
};
```

## Setting a custom appearance

You can customize the appearance of the onboarding flow by passing an `appearance` object to the `init` method.

```typescript
const component = footprint.init({
  publicKey: "pb_test_LRfe5D3dZnWSf5vOGSeCFk",
  onComplete: (validationToken) => {
    console.log(validationToken);
  },
  appearance: {
    variables: {
      borderRadius: "8px",
      colorSuccess: "#10b981",
      colorError: "#F87171",
      buttonPrimaryBg: "#5550e9",
    },
  },
});

component.render();
```

For more information, including a list of available variables, check out the [customization guide](/integrate/customization).

## Available Props

| Variable        | Description                                                                                                                                                                                                                     |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `publicKey`     | Optional. Onboarding configuration public key.                                                                                                                                                                                  |
| `authToken`     | Optional. A valid string authentication token generated using the Secret API Key, Footprint User ID and fields to collect. One of publicKey or authToken must be provided. More info [here](/integrate/user-specific-sessions). |
| `redirectUrl`   | Required. The deep link so that the user can go back to your app after finishing the flow. E.g: footprint://callback/                                                                                                           |
| `onComplete`    | Optional. Triggered after the user completes the onboarding flow. You'll receive a `validationToken` that your backend can exchange with Footprint to see the fp_id, the login method used, and the KYC status.                 |
| `onError`       | Optional. A function that is called there was an unrecoverable error while initializing the onboarding flow. It takes in an error string argument with more details.                                                            |
| `onCancel`      | Optional. Triggered when the user abandons the flow. This can be triggered when the user clicks on the close button inside our iframe                                                                                           |
| `onClose`       | Optional. Triggered when the user closes the flow (either completed or canceled).                                                                                                                                               |
| `bootstrapData` | Optional. An object that contains previously collected user data that can bootstrap the onboarding flow. More information [here](/integrate/bootstrap-data).                                                                    |
| `options`       | Optional. An options object that customizes the flow, can accept `showLogo` and `showCompletionPage` entries. More info [here](/integrate/customization#available-options)                                                      |
| `appearance`    | Optional. An object that customizes the look of your integration. More info [here](/integrate/customization)                                                                                                                    |
| `l10n`          | Optional. Specifies the desired localization. More information [here](/integrate/customization#localization-configuration).                                                                                                     |
