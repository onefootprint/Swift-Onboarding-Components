## 1. What are onboarding components

Our [Flutter hosted flow](/articles/sdks/flutter-hosted) provides a handy way to pass pre-existing bootstrap data and appearance styling to open the verification/onboarding flow on a web view. However, many developers prefer to incorporate data collection forms directly within their apps, customized to their specific design needs. To address this, our Flutter Onboarding Components SDK provides a suite of versatile UI widgets that can be seamlessly embedded into apps. These widgets are designed to handle essential data collection and validation logic for the Footprint onboarding flow internally while giving developers full control over the appearance and user experience. The SDK also provides a set of necessary utility functions to identify users, save data into Footprint vault, etc.

## 2. Installation

Please follow `footprint_flutter` installation instructions [here](/articles/sdks/flutter-introduction#installation).

## 3. Getting started

### 3.1 Get a public key

- Go to the [playbooks page](https://dashboard.onefootprint.com/playbooks) on your dashboard.
- Use an existing playbook or create a new one.
- Obtain the Publishable Key, e.g., `pb_test_GfF2M0HxXQwrcB3ETl0yhe`.
  <br />
  <br />

### 3.2 Add Footprint Provider

To use Onboarding Components, you need to start by wrapping all of you onboarding-related widgets with the `FootprintProvider` widget. This widget provides the necessary context for the Footprint components to work. `FootprintProvider` constructor has three required arguments that you must pass - `String publicKey` , `String redirectUrl` , `Widget child` .

```dart highlight="2,19, 20, 21, 22, 23"
import 'package:flutter/material.dart';
import 'package:footprint_flutter/footprint_flutter.dart';

void main() {
  runApp(const MyApp());
}

class MyApp extends StatelessWidget {
  const MyApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      home: Builder(
        builder: (_) => Scaffold(
          appBar: AppBar(
            title: const Text('Footprint Flutter Demo'),
          ),
          body: FootprintProvider(
            publicKey: "YOUR PUBLIC KEY",
            redirectUrl: "YOUR REDIRECT URL",
            child: Text("A widget containing the onboarding components."),
          ),
        ),
      ),
    );
  }
}
```

## 5. Identify your user

With the Provider in place, you can now start using Onboarding components. Let's start by identifying the user. You can identify a user using authToken (that you pass to the Provider ) or email/phone that you collect from user during the identification process. We support creating auth flow inline as well as doing the auth on a webview. We suggest using the webview auth as a backup for rare cases when inline auth is not supported. Here is how you can build auth flow using our SDK.

### 5.1 Check auth requirements

We start by calling the function `requiresAuth()` . This function needs to be called only once and after the first frame is render. So, we put it inside `initState()` in a callback to `WidgetsBinding.instance.addPostFrameCallback` . `requiresAuth` will return `({bool requiresAuth, VerificationResult? verificationResult})` .

- `requiresAuth` will tell you if you need to authenticate at all. If you provided a fully scoped `authToken` , you will most likely not need to authenticate again.
- The `VerificationResult? verificationResult` will provide you a record `({Fields? fields, Requirements? requirements, String validationToken, FormData? vaultData})`.
  - You can use the validation token to find the `fp_id` using [this API](/api-reference#post-onboarding-session-validate). `VerificationResult` will be `null` if the `requiresAuth` prop above returns `true` i.e. if you need to authenticate.
  - `Fields? fields` contains `({List<DataIdentifier> missing, List<DataIdentifier> collected, List<DataIdentifier> optional})` to help you guide through the fields you need to collect and the ones that has already been collected.
  - `Requirements? requirements` more contexts on requirements for the flow such as whether data-collection part is done, or id-doc is required, etc.
  - `FormData? vaultData` is an object containing the decrypted data that has already been vaulted before this flow started possibly (but not always) using API or in an incomplete flow.

Note that you can get the updated `Fields` and `Requirements` in the middle of the flow by calling `footprintUtils(context).getRequirements()`.

_Note: You can skip calling this function if you didn’t provide auth token and authenticate using email and phone._

### 5.2 Create challenge

Next we create challenge based on the response from the previous step. We have a widget, few helper functions and values to accomplish this.

- `footprintUtils(context).isReadyForAuth` tells you whether the SDK is ready to create an auth challenge.
- `FootprintOtp` is a widget that provides a set of OTP utils: `createAuthTokenBasedChallenge` , `createEmailPhoneBasedChallenge` , and `verifyOtpChallenge` (we will talk about verification in the next step).
- In rare cases when we don’t support inline OTP, we throw `InlineOtpNotSupportedException` . You can use `footprintUtils(context).launchIdentify` to launch the OTP flow on the webview in those cases.

Additionally, to collect email and phone from the user, we need to create a form with the user's email and phone number using a set of form, field, and input widgets we provide with our SDK. Our widgets are raw, meaning that you have to provide necessary styles to match your application's design.

class _IdentifyState extends State<Identify> {
  bool isChallengeCreated = false;
  ChallengeKind? challengeKind;
  var requiresAuth;

  @override
  void initState() {
    super.initState();
    WidgetsBinding.instance.addPostFrameCallback((_) {
      footprintUtils(context).requiresAuth().then((response) {
        if (response.requiresAuth == false) {
          widget.handleAuthenticated(response.verificationResult);
        } else {
          setState(() {
            requiresAuth = response.requiresAuth;

          return const CircularProgressIndicator();
        },
      );

                    onPressed: () {

## 6. Collecting user’s data

Once the authentication is completed, you can start collecting user data.

### 6.1 Data collection form

The data collection form is constructed similarly to the identification form.

## 7. Finalizing the onboarding

Once all the data are collected, we need to finalizing the onboarding. We can verify if all the data are collected by checking the response from `footprintUtils(context).getRequirements()`. In order to finalize the onboarding, we have two options:

- If all requirements are completed and there is no passkey and id-doc requirements, we can call `footprintUtils(context).process()` for process the onbaording inline. `footprintUtils(context).process()` return a validation token.
- `footprintUtils(context).process()` will throw an `InlineProcessException` if all requirements are not completed or if inline process is not supported for this case. If that happens you can handoff to Footprint's flow to finalize the onboarding: we'll collect any missing data, as well as register passkeys (if needed) and finalize the onboarding sending you a validation token. You can handoff to Footprint’s flow calling the `handoff` function which can be accessed by calling `footprintUtils` function.

```dart highlight="91,92,93, 95, 96, 97, 98, 99, 100, 101, 102, 103, 104, 105, 106, 107, 324, 325, 326, 327, 328, 329, 330, 331, 332, 333, 334, 335, 336"
    });
  }

  void handleVerfied(VerificationResult? verificationResult) {
    widget.handleAuthenticated(verificationResult);
  }

  @override
  Widget build(BuildContext context) {
    if (requiresAuth == null) {
      return const Center(child: CircularProgressIndicator());
    }

    if (widget.useAuthToken) {
      return FootprintOtp(
        buildOtp: (otpUtils) {
          if (!isChallengeCreated) {
            otpUtils.createAuthTokenBasedChallenge().then((challengeKind) {
              handleChallengeCreated(challengeKind);
            }).catchError(
              (_) {
                footprintUtils(context).launchIdentify(
                    onAuthenticated: (verificationResult) {
                  widget.handleAuthenticated(verificationResult);
                } // Don't pass email and phone number - it's going to use auth token
                    );
              },
              test: (err) => err is InlineOtpNotSupportedException,
            );
            return const Center(child: CircularProgressIndicator());
          }

          return Container(
            padding: const EdgeInsets.fromLTRB(48, 20, 48, 20),
            child: TextField(
              decoration: inputDecoration(
                "Enter OTP from ${challengeKind == ChallengeKind.sms ? 'SMS' : 'Email'}",
              ),
              onSubmitted: (value) {
                otpUtils
                    .verifyOtpChallenge(verificationCode: value)
                    .then((verificationResult) {
                  handleVerfied(verificationResult);
                });
              },
            ),
          );
        },
      );
    }

    return FootprintOtp(
      buildOtp: (otpUtils) {
        if (!isChallengeCreated) {
          return FootprintForm(createForm: (handleSubmit, props) {
            return Container(
              padding: const EdgeInsets.fromLTRB(48, 20, 48, 20),
              alignment: Alignment.center,
              child: Column(
                children: [
                  Text("Identification",
                      style: Theme.of(context).textTheme.titleMedium),
                  const SizedBox(height: 16),
                  FootprintField(
                    name: DataIdentifier.idEmail,
                    createField: ({error}) {
                      return FootprintTextInput(
                        labelText: "Email",
                        decoration:
                            inputDecoration("Email", errorText: error?.message),
                      );
                    },
                    // NOTE: Alternatively, you can use the `child` property to pass in a widget (example below for phone number)
                  ),
                  const SizedBox(height: 12),
                  FootprintField(
                    name: DataIdentifier.idPhoneNumber,
                    child: FootprintTextInput(
                      labelText: "Phone Number",
                      decoration: inputDecoration("Phone Number"),
                    ),
                  ),
                  const SizedBox(height: 12),
                  ElevatedButton(
                    onPressed: () {
                      handleSubmit();
                    },
                    child: const Text('Submit'),
                  ),
                ],
              ),
            );
          }, onSubmit: (formData) {
            otpUtils
                .createEmailPhoneBasedChallenge(
              email: formData.email ?? '',
              phoneNumber: formData.phoneNumber ?? "",
            )
                .then((challegeKind) {
              handleChallengeCreated(challegeKind);
            }).catchError(
              (_) {
                footprintUtils(context).launchIdentify(
                  email: formData.email,
                  phoneNumber: formData.phoneNumber,
                  onAuthenticated: (verificationResult) {
                    widget.handleAuthenticated(verificationResult);
                  },
                );
              },
              test: (err) => err is InlineOtpNotSupportedException,
            );
          });
        }
        return Container(
          padding: const EdgeInsets.fromLTRB(48, 20, 48, 20),
          child: TextField(
            decoration: inputDecoration(
              "Enter OTP from ${challengeKind == ChallengeKind.sms ? 'SMS' : 'Email'}",
            ),
            onSubmitted: (value) {
              otpUtils
                  .verifyOtpChallenge(verificationCode: value)
                  .then((verificationResult) {
                handleVerfied(verificationResult);
              });
            },
          ),
        );
      },
    );
  }
}

class DataCollection extends StatelessWidget {
  const DataCollection({super.key, required this.onCompleted});

  final void Function(String validationToken) onCompleted;

  void handleComplete(BuildContext context, FormData formData) {
    footprintUtils(context)
        .vault(
      formData,
    )
        .then(
      (_) {
        footprintUtils(context).process().then((vTok) {
          onCompleted(vTok);
        }).catchError(
          (error) {
            footprintUtils(context).handoff(
              onComplete: (validationToken) => onCompleted(validationToken),
              onCancel: () => print("Handoff Cancelled"),
              onError: (err) => print("Handoff Error $err"),
            );
          },
          test: (error) => error is InlineProcessException,
        );
      },
    ).catchError(
      (err) {
        print("Save Error $err");
      },
    );
  }

  @override
  Widget build(BuildContext context) {
    return FootprintForm(
      createForm: (handleSubmit, props) {
        return Padding(
          padding: const EdgeInsets.all(20.0),
          child: Container(
            padding: const EdgeInsets.fromLTRB(48, 20, 48, 20),
            alignment: Alignment.center,
            child: Column(
              children: [
                Text("Basic Information",
                    style: Theme.of(context).textTheme.titleMedium),

      onSubmit: (formData) {

  }
```
