When integrating with Footprint, you may want your application to receive events as they occur in Footprint so your backend system can react accordingly.

To enable webhook events, you need to register webhook endpoints in the [Footprint dashboard](https://dashboard.onefootprint.com/webhooks). After you register them, Footprint can push real-time event data to your application’s webhook endpoint when events happen in Footprint. Footprint uses HTTPS to send webhook events to your app as a JSON payload.

# Getting started

To get started, see the [guide to consuming webhooks](https://docs.svix.com/receiving/introduction).

<inline-alert variant="info">
  We recommend that you secure your integration by always verifying that all

  verifying webhook
  signatures](https://docs.svix.com/receiving/verifying-payloads/how).

# Recommended integration

When integrating with Footprint's frontend SDKs for collecting and verifying user information, you most commonly may want to subscribe to events related to a user's status.

## Onboarding completed

We recommend subscribing to the <navigation-link fontFamily="code" href="https://www.svix.com/event-types/us/org_2LbAthqjvv71cc7eYpwvHoeuuEw/#footprint.onboarding.completed">footprint.onboarding.completed</navigation-link> event, which will fire when a user's onboarding has reached a terminal status.

In most cases, the [status received synchronously](/kyc/getting-started#verify-the-footprint-token-server-side-check-the-onboarding-status) at the end of Footprint's onboarding flow will be a terminal `pass`, `fail`, or `none`. In some cases where identity verifications vendors take longer to verify a user, you will synchronously receive a `pending` status in your integration with the frontend SDK. In these cases, you will receive the final decision for an onboarding through the `footprint.onboarding.completed` event.

## Manual review

Employees at your company may change a user's status while manually reviewing a user in the Footprint dashboard. This will fire the <navigation-link fontFamily="code" href="https://www.svix.com/event-types/us/org_2LbAthqjvv71cc7eYpwvHoeuuEw/#footprint.user.manual_review">footprint.user.manual_review</navigation-link> event. Upon receiving this webhook, we recommend consulting the <navigation-link fontFamily="code" href="/api-reference#get-users-fp-id">GET /users/{fp_id}</navigation-link> API for information on the user's new manual status.

## Watchlist checks

If your playbook has continuous monitoring enabled, Footprint will regularly check if any of your users are found on AML watchlists. Updates on watchlist checks are sent using the <navigation-link fontFamily="code" href="https://www.svix.com/event-types/us/org_2LbAthqjvv71cc7eYpwvHoeuuEw/#footprint.watchlist_check.completed">footprint.watchlist_check.completed</navigation-link> event.
