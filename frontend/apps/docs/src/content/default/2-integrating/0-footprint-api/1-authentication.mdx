The Footprint API uses API keys to authenticate requests. You can view and manage your API keys in the [developer dashboard](https://dashboard.onefootprint.com/developers).

| Prefix    | Description                                                                        |
| --------- | ---------------------------------------------------------------------------------- |
| `sk_test` | Prefix for **Sandbox** mode API keys. No costs are incurred when using these keys. |
| `sk_live` | Prefix for **Live** mode API keys                                                  |

<inline-alert variant='warning'>Treat this key like a sensitive password! Your API keys carry many privileges, so be sure to keep them secure! Do not post your secret API keys in publicly accessible places like GitHub or client-side code like web front-ends.</inline-alert>

### Basic Authentication

Authentication to the API is performed via [HTTP Basic Authentication](https://developer.mozilla.org/en-US/docs/Web/HTTP/Authentication). Provide your API key as the basic auth username value with no password. In `curl` syntax this would be written as `-u <api_key>:`. For example:

```bash
curl https://api.onefootprint.com/users -u sk_test_CXUsbCR8j2kH6e5GeEl8eSBnQTIPCUaKpv:
```

### Header Authentication

Alternatively, Footprint supports header-based auth via the `X-Footprint-Secret-Key` header. For example, you can authenticate with an HTTP header using cURL:

```bash
curl https://api.onefootprint.com/users -H "X-Footprint-Secret-Key: sk_test_CXUsbCR8j2kH6e5GeEl8eSBnQTIPCUaKpv"
```

<inline-alert variant='info'>All API requests must be made over HTTPS, calls made over plain HTTP will fail.</inline-alert>
