In a context where a user has already authenticated to your application, you may invoke Footprint.js with an auth token specific to that user. This allows you to progressively collect new and updated information from existing users, and essentially bootstraps onboarding flows using the information you've already collected about a user.

## Prerequisites

Please read our [Server-side API Authentication guide](/integrate/api-auth).

## Step 1: Get or create a Footprint user on your backend

You will likely store a mapping from your own user records to Footprint's identifier for the user, the `fp_id`. When the user logs into your application, look up the specific `fp_id` for the user for whom you'd like to open an onboarding flow.

If your user doesn't have an `fp_id` yet, create a new user in Footprint and vault any information you've already collected from them. More information can be found [here](/integrate/migrate-existing-data).

Data already in the user's Footprint vault won't need to be recollected from the user when opening a user-specific onboarding flow, but they may edit it.

## Step 2: Generate an auth token for the user on your backend

From your backend, using a secret API key, generate an auth token associated with the specific user using the <navigation-link href="/api-reference#post-users-fp-id-token" fontFamily="code">POST /users/{fp_id}/token</navigation-link> API:

### Example request

```bash
curl https://api.onefootprint.com/users/fp_id_K0q6Eh6Rr3WOOfFBLPiHsr/token \
  -X POST \
  -d '{"kind": "onboard", "key": "pb_live_qBV2iO3nQI5moqo5GJlFhI"}' \
  -u sk_test_CXUsbCR8j2kH6e5GeEl8eSBnQTIPCUaKpv:
```

### Example response

```json
{
  "token": "tok_vJK5Ze2N5fQ1GtE5V770BH8CZtQwXHF1hxowB9Nowh0",
  "expires_at": "2023-10-24T20:44:13.639341Z"
}
```

### Token kinds

You are able to create auth tokens for the user that have different purposes.

| Token kind  | Description                                                                                                                                             |
| ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `onboard`   | Onboard the user onto a specific playbook, specified by the `key` (from the [Playbooks tab](https://dashboard.onefootprint.com/playbooks)).             |
| `reonboard` | Convenience method to reonboard the user onto the last playbook that they onboarded onto.                                                               |
| `inherit`   | Inherit any operation previously requested via the dashboard, including reonboarding, ID document collection, and SSN card collection. More info below. |
| `user`      | Simply create a token for the user. A playbook key may be provided directly to the Footprint Verify SDK to trigger onboarding.                          |

You can find more info in the <navigation-link href="/api-reference#post-users-fp-id-token" fontFamily="code">POST /users/{fp_id}/token</navigation-link> API reference.

### The inherit token kind

The `inherit` token kind is powerful and deserves more explanation. On Footprint's admin dashboard, you are able to request additional information from a user who has already onboarded onto your service. For example, you may request the user to upload an SSN card or a driver's license. By default, this will send a link to the user to allow them to complete the form.

If you would prefer a more native experience to your application, you can instead embed the Footprint Verify component to complete any outstanding data requests.

Before generating an `inherit` token and invoking the Footprint Verify SDK for the user, you can check via API if a user has an outstanding request to provide more info using the <navigation-link fontFamily="code" href="/api-reference#get-users-fp-id">GET /users/{fp_id}</navigation-link> API:

#### Example request

```bash
curl https://api.onefootprint.com/users/fp_id_K0q6Eh6Rr3WOOfFBLPiHsr \
  -u sk_test_CXUsbCR8j2kH6e5GeEl8eSBnQTIPCUaKpv:
```

#### Example response

```json
{
  "id": "fp_id_K0q6Eh6Rr3WOOfFBLPiHsr",
  "requires_manual_review": false,
  "status": "pass",
  "requires_additional_info": {
    "timestamp": "2023-12-12T21:28:38.771377Z",
    "note": "Hi Christian, we can't wait for you to get started with your Acme Bank credit card! To finish verifying your identity, can you please submit a photo of your SSN card? Once received, we can approve your application and mail out your credit card."
  }
}
```

If the `requires_additional_info` field is non-null, there is an outstanding request to collect information from this user, and you can create an `inherit` token. It will also include a human-readable note you provided in the dashboard to display to your user, which you may choose to render in your own application.

## Step 3: Pass auth token into Footprint Verify SDKs

You can provide this auth token to the Footprint Verify SDK. This will ask the user to authenticate and then proceed with collecting any outstanding information specified in the playbook. Check out the specific documentation for the SDK you're using.

If you are using our React integration, you can pass the `authToken` as a prop into the `FootprintVerifyButton` component:

```typescript
import '@onefootprint/footprint-js/dist/footprint-js.css';
import { FootprintVerifyButton } from '@onefootprint/footprint-react';

const MyComponent = () => {
  const handleComplete = (validationToken: string) => {
    // TODO: Post to Footprint server to fetch user ID or verification status
  };

  return (
    <FootprintVerifyButton
      authToken='tok_vJK5Ze2N5fQ1GtE5V770BH8CZtQwXHF1hxowB9Nowh0'
      onComplete={handleComplete}
    />
  );
};

export default MyComponent;
```

If you are using our Javascript integration, you can pass the `authToken` along with the other props into the `footprint.init` method:

```javascript
import '@onefootprint/footprint-js/dist/footprint-js.css';
import footprint from '@onefootprint/footprint-js';

const handleClick = () => {
  const component = footprint.init({
    kind: 'verify',
    authToken: 'tok_vJK5Ze2N5fQ1GtE5V770BH8CZtQwXHF1hxowB9Nowh0', // Auth token from Step 2
    onComplete: validationToken => {
      // TODO: Post to Footprint server to fetch user ID or verification status
    },
  });
  component.render();
};
```
