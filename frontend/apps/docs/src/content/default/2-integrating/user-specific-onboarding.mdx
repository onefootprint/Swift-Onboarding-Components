In a context where a user has already authenticated to your application, you may invoke Footprint.js with an auth token specific to that user. This allows you to progressively collect new and updated information from existing users. It essentially bootstraps and onboarding flows using the information you've already collected about a user.

## Prerequisites

Please read our [Server-side API Authentication guide](/integrate/api-auth).

## Step 1: Get or create a Footprint user on your backend

You will likely store a mapping from your own user records to Footprint's identifier for the user, the `fp_id`. Look up the specific `fp_id` for the user for whom you'd like to open an onboarding flow.

If your user doesn't have an `fp_id` yet, create a new user in Footprint and vault any information you've already collected from them. More information can be found [here](/integrate/migrate-existing-data).

Data already in the user's Footprint vault won't need to be recollected from the user when opening a user-specific onboarding flow, but they may edit it.

## Step 2: Generate an auth token for the user on your backend

From your backend, using a secret API key, generate an `authToken` associated with the specific `fp_id`. Select the playbook you'd like the user to onboard onto, and provide its playbook publishable key (from the [Playbooks tab](https://dashboard.onefootprint.com/playbooks)).

### Example request

```bash
curl https://api.onefootprint.com/users/fp_id_K0q6Eh6Rr3WOOfFBLPiHsr/token \
  -X POST \
  -d '{"key": "ob_live_qBV2iO3nQI5moqo5GJlFhI"}' \
  -u sk_test_CXUsbCR8j2kH6e5GeEl8eSBnQTIPCUaKpv:
```

### Example response

```bash
{
  "token": "tok_vJK5Ze2N5fQ1GtE5V770BH8CZtQwXHF1hxowB9Nowh0",
  "expires_at": "2023-10-24T20:44:13.639341Z"
}
```

You can find more info in the [API reference](/api-reference#post-users-token).

## Step 3: Pass auth token into Footprint Verify SDKs

If you are using our React integration, you can pass the `authToken` as a prop into the `FootprintVerifyButton` component:

```typescript
import '@onefootprint/footprint-js/dist/footprint-js.css';
import { FootprintVerifyButton } from '@onefootprint/footprint-react';

const MyComponent = () => {
  const handleComplete = (validationToken: string) => {
    // TODO: Post to Footprint server to fetch user ID or verification status
  };

  return (
    <FootprintVerifyButton
      authToken='tok_vJK5Ze2N5fQ1GtE5V770BH8CZtQwXHF1hxowB9Nowh0'
      onComplete={handleComplete}
    />
  );
};

export default MyComponent;
```

If you are using our Javascript integration, you can pass the `authToken` along with the other props into the `footprint.init` method:

```javascript
import footprint from '@onefootprint/footprint-js';

const handleClick = () => {
  const component = footprint.init({
    kind: 'verify',
    authToken: 'tok_vJK5Ze2N5fQ1GtE5V770BH8CZtQwXHF1hxowB9Nowh0' // Auth token from Step 2
    onComplete: (validationToken) => {
      // TODO: Post to Footprint server to fetch user ID or verification status
    }
  });
  component.render();
};
```

Since you provided the playbook publishable key when creating the auth token, it cannot be provided to Footprint.js.
