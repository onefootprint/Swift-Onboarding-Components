## Introduction

If you already have certain data about your user, you can share it with Footprint to expedite the onboarding process for your users.
We will pre-fill already available data and skip completed pages for your users. The users can always edit and correct this data
while onboarding through Footprint. To pass pre-collected data to Footprint's flow, use the `bootstrapData` option in the SDK.

Bootstrap data will be validated using the same validations applied to user-entered data. Any bootstrap data that fails validation will be ignored and the user will be asked to re-provide the value.

Note also that only data fields that are collected by the provided playbook may be provided as bootstrap data.

## Boostraping KYC data

<inline-alert variant='info'>Check out [user-specific onboarding](/articles/integrate/user-specific-sessions) which allows bootstrapping data using a new or existing Footprint user. If you already have user data stored on your backend, you may want to create a Footprint user with this data and start a user-specific onboarding session.</inline-alert>

```typescript
import "@onefootprint/footprint-js/dist/footprint-js.css";
import footprint from "@onefootprint/footprint-js";

const launchFootprint = () => {
  const component = footprint.init({
    kind: "verify",
    publicKey: "pb_test_VMooXd04EUlnu3AvMYKjMW",
    bootstrapData: {
      "id.email": "jane.doe@acme.com",
      "id.phone_number": "+12025550179",
      "id.first_name": "Jane",
      "id.middle_name": "Samantha",
      "id.last_name": "Doe",
      "id.dob": "01/01/2001",
      "id.ssn9": "123456789",
      "id.nationality": "US",
      "id.address_line1": "123 Apple St.",
      "id.address_line2": "APT 123",
      "id.city": "Boston",
      "id.state": "MA",
      "id.country": "US",
      "id.zip": "02117",
    },
  });
  component.render();
};
```

### Available user data

| Variable                  | Description                                                                                                                       |
| ------------------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| `id.email`                | User email (e.g `jane.doe@acme.com`)                                                                                              |
| `id.phone_number`         | User phone. Must be in the [E.164](https://www.twilio.com/docs/glossary/what-e164) format (e.g `+12025550179`)                    |
| `id.first_name`           | User's first name.                                                                                                                |
| `id.middle_name`          | User's middle name.                                                                                                               |
| `id.last_name`            | User's last name.                                                                                                                 |
| `id.dob`                  | User's date of birth. Must be in MM/DD/YYYY format. (e.g. 01/01/2001)                                                             |
| `id.ssn9`                 | User's 9-digit SSN number. (e.g. 123456789)                                                                                       |
| `id.ssn4`                 | The last 4 digits of user's SSN. (e.g. 1234)                                                                                      |
| `id.address_line1`        | The first line of user's residential address.                                                                                     |
| `id.address_line2`        | Additional address information, like apartment or suite number. Must be a string.                                                 |
| `id.city`                 | User's residential city. Must be a string.                                                                                        |
| `id.state`                | User's residential state. Must be in 2-character [USPS](https://pe.usps.com/text/pub28/28apb.htm) format. (e.g. CA)               |
| `id.country`              | User's residential country. Must be in [Alpha-2 Code](https://www.iban.com/country-codes) format. (e.g. US)                       |
| `id.zip`                  | User's residential zip. Must be a string.                                                                                         |
| `id.us_legal_status`      | User's legal status in the U.S. Must be one of `'citizen'`, `'permanent_resident'`, or `'visa'`.                                  |
| `id.nationality`          | User's nationality. Must be in [Alpha-2 Code](https://www.iban.com/country-codes) format. (e.g. US)                               |
| `id.citizenships`         | User's citizenships. Must be an array of [Alpha-2 Codes](https://www.iban.com/country-codes). (e.g. `['BR', 'IT', 'HK']`)         |
| `id.visa_kind`            | Type of visa the user has. Must be one of `'e1'`, `'e2'`, `'e3'`, `'f1'`, `'g4'`, `'h1b'`, `'l1'`, `'o1'`, `'tn1'`, or `'other'`. |
| `id.visa_expiration_date` | User's visa expiration date. Must be in MM/DD/YYYY format. (e.g. 01/01/2001)                                                      |

<inline-alert variant='info'>Note, any bootstrap attributes that are not collected by the provided playbook will be ignored.</inline-alert>

## Boostraping KYB data

```typescript
import "@onefootprint/footprint-js/dist/footprint-js.css";
import footprint from "@onefootprint/footprint-js";

const launchFootprint = () => {
  const component = footprint.init({
    kind: "verify",
    publicKey: "pb_test_VMooXd04EUlnu3AvMYKjMW",
    bootstrapData: {
      "business.name": "Acme Bank",
      "business.dba": "Acme",
      "business.tin": "12-3456789",
      "business.website": "www.google.com",
      "business.phone_number": "+12025550179",
      "business.address_line1": "123 Main St",
      "business.address_line2": "Apt 123",
      "business.city": "Boston",
      "business.state": "MA",
      "business.zip": "02117",
      "business.country": "US",
      "business.corporation_type": "c_corporation",
    },
  });
  component.render();
};
```

### Available business data

| Variable                    | Description                                                                                                                                             |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `business.name`             | Name of the business                                                                                                                                    |
| `business.dba`              | Doing-business-as alias                                                                                                                                 |
| `business.tin`              | Taxpayer identification number, 9-digit string (hyphens will be ignored and stripped out)                                                               |
| `business.website`          | Website of the business                                                                                                                                 |
| `business.phone_number`     | Phone number                                                                                                                                            |
| `business.address_line1`    | First line of the business's street address                                                                                                             |
| `business.address_line2`    | Second line of the business's street address                                                                                                            |
| `business.city`             | City of the business's street address                                                                                                                   |
| `business.state`            | State of the business's street address                                                                                                                  |
| `business.zip`              | Postal code for the business's street address                                                                                                           |
| `business.country`          | Country for the business's street address                                                                                                               |
| `business.corporation_type` | One of `c_corporation`, `s_corporation`, `b_corporation`, `llc`, `llp`, `partnership`, `sole_proprietorship`, `non_profit`, `unknown`, `trust`, `agent` |

<inline-alert variant='info'>Note, any bootstrap attributes that are not collected by the provided playbook will be ignored.</inline-alert>
