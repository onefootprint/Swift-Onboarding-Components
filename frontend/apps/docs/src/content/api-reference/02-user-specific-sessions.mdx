# User-specific sessions

Footprint supports a rich set of onboarding flows that you can invoke for existing users. When a user is already in a logged-in context in your application, you may want to launch an additional Footprint flow (like collecting and verifying a driver's license image) for the specific user that is logged in.

You can find much more information on user-specific sessions in the documentation site [here](/articles/integrate/user-specific-sessions).
