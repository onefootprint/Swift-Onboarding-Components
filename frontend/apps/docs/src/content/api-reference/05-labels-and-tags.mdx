  path: /users/{fp_id}/tags
  title: Add a tag for a user
  description: Adds a new free-form tag to the provided user.
- method: get
  path: /users/{fp_id}/tags
  title: List tags for a user
  description: Lists the free-form tags on the provided user.
- method: delete
  path: /users/{fp_id}/tags/{tag_id}
  title: Delete a tag for a user
  description: Delete the provided free-form tag from the provided user.

# Labels and tags

In order to help you visualize your user base and improve the verification decisions that Footprint makes for your users, we support a set of interactive APIs to deliver feedback on users onboarded through Footprint based on context specific to your application. We support this feedback in two ways:
- Labels are a way to provide a high-level determination about a user, and to give feedback to Footprint to improve risk systems and detection. You can have 1 active Label per user
- Tags are an unstructure way for you to organize, categorize and add metadata about your users. A user can have multiple tags
